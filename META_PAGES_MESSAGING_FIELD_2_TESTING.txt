**Step-by-Step Testing Guide:**

**Prerequisites:**
1. A Facebook Page with an Instagram Business account connected
2. The test user must have admin or editor role on the page
3. The app must be authorized with all required permissions, including pages_messaging

**Step 1: Authorize the App**
- Navigate to: https://{frontend-url}/onboard/meta
- Click "Login with Facebook"
- Grant all requested permissions, including pages_messaging
- Select the Facebook Page and Instagram Business account to connect
- Complete the authorization process

**Step 2: Verify Webhook Subscription**
- After account linking, the app automatically subscribes to page webhooks
- Verify webhook subscription status by calling: GET /instagram/accounts/{customer_id}/webhook-status
- Expected response: `webhook_subscribed: true` with `subscribed_fields` including `messages`
- This demonstrates the use of pages_manage_metadata permission (for webhook subscription)

**Step 3: Send a Test Message**
- From a personal Instagram account (different from the test account), send a Direct Message to the connected Instagram Business account
- The message should be received by the app's webhook endpoint within seconds
- This demonstrates the pages_messaging permission is working for receiving messages

**Step 4: Verify Message Reception**
- Check the app's dashboard or logs to confirm the message was received
- The app should display the incoming message and can trigger automated responses
- This verifies the webhook subscription is functioning correctly

**Step 5: Send a Response Message**
- Use the app's interface to send a response message to the customer
- The message should be delivered to the customer's Instagram account
- Verify the message appears in the customer's Instagram Direct Messages
- This demonstrates the pages_messaging permission is working for sending customer support messages

**Step 6: Verify Complete Messaging Workflow**
- Confirm the complete workflow: receive message → process → send response
- Verify that automated responses (if configured) are working correctly
- This demonstrates the full functionality of the pages_messaging permission

---

**Testing Checklist:**

✅ App authorization with pages_messaging permission
✅ Webhook subscription successful (pages_manage_metadata usage)
✅ Message reception via webhook (pages_messaging usage)
✅ Message sending via Instagram Messaging API (pages_messaging usage)
✅ Automated response system (if configured)
✅ Complete messaging workflow functioning
